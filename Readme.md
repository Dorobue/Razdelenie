Partition Variance Minimization
Описание задачи

Проект посвящён задаче разбиения последовательности чисел на n подряд идущих групп таким образом, чтобы дисперсия сумм групп была минимальной.

Формальная постановка:

задан список чисел
L = [x₁, x₂, ..., xₘ],

требуется разбить его на n непустых последовательных подсписков,

для каждого подсписка считается сумма,

минимизируется дисперсия полученных сумм.

Метрика качества

Для оценки качества разбиения используется дисперсия:

D = E[X²] − (E[X])²


где X — суммы подсписков.

Чем меньше D, тем более равномерно распределены элементы между группами.

Реализованные алгоритмы
1. Стохастический алгоритм (razdelrandom)

Случайным образом генерируются допустимые разбиения, из которых выбирается лучшее.

Особенности:

число итераций s выбирается как 10^n при n ≤ 6, иначе 10^6;

не гарантирует нахождение глобального минимума;

качество решения резко падает при росте размерности задачи;

время работы быстро растёт с увеличением n.

2. Полный перебор по позициям разрывов (razdel_plus)

Генерируются все возможные комбинации позиций разбиений между элементами списка.

Особенности:

гарантирует нахождение оптимального решения;

существенно быстрее битового перебора;

имеет комбинаторную сложность;

не масштабируется по памяти при больших входных данных.

3. Полный перебор битовых масок (razdel_plus01)

Перебор всех двоичных масок длины len(list) с фиксированным числом разрывов.

Особенности:

концептуально простой метод;

экспоненциальная сложность O(2^m);

на практике самый медленный алгоритм.

Экспериментальные данные

Тестирование проводилось на:

малых списках (10 элементов),

средних списках (20 элементов),

большом списке (150 элементов, случайные значения).

Для каждого списка рассматривались значения:

n = 4,

n = 8.

Измерялись:

значение дисперсии D,

время выполнения алгоритма.

Критический эксперимент (150 элементов)
Входные данные

длина списка: 150 элементов

число групп: 8

Результаты
Стохастический метод
D = 125406.0
Time ≈ 81.39 s


Качество решения крайне низкое — стохастический алгоритм не приближается к оптимальному разбиению даже за длительное время работы.

Полный перебор по позициям разрывов

При запуске алгоритма происходит аварийное завершение:

MemoryError


Причина — экспоненциальный рост числа комбинаций, приводящий к переполнению памяти при рекурсивной генерации всех вариантов разбиений.

Вывод по эксперименту

Стохастический метод формально отрабатывает, но даёт практически бесполезный результат.

Полный перебор теоретически корректен, но не применим на практике для больших входных данных из-за ограничений по памяти.

Задача требует либо:

более сильных эвристик,

либо динамического программирования,

либо приближённых алгоритмов с контролем ошибки.

Основные выводы

Для малых входных данных все методы находят одинаковый минимум дисперсии.

Стохастический алгоритм не масштабируется ни по времени, ни по качеству решения.

Полный перебор по разрывам является оптимальным по качеству, но ограничен по памяти.

Эксперимент на 150 элементах демонстрирует практическую неразрешимость задачи полным перебором.

Ограничения реализации

Используется глобальная переменная greatlist.

Рекурсивная генерация приводит к переполнению стека и памяти.

Замеры времени выполняются через time.time().

Назначение проекта
